# Ascend SFA v1 重构软件设计说明书

## 1. 概述

### 1.1 项目背景
当前 `sfa_v1.py` 文件中的 SFA (Sparse Flash Attention) 实现将 prefill 和 decode 过程混合在一个统一的 `forward` 方法中，导致代码逻辑复杂、可维护性差。参考 `mla_v1.py` 的成功实现，需要对 SFA 进行重构，将 prefill 和 decode 过程明确拆分。

### 1.2 重构目标
- 将 prefill 和 decode 的注意力计算逻辑完全分离
- 创建专门的 prefill 和 decode 元数据结构
- 实现独立的 prefill 和 decode 预处理方法
- 提高代码的可读性、可维护性和可测试性

## 2. 核心设计原则

### 2.1 与 MLA 对齐
- 保持与 `mla_v1.py` 相同的设计模式
- 统一的元数据结构命名和组织方式
- 一致的 prefill/decode 判断和处理逻辑

### 2.2 向后兼容
- 保持现有的 API 接口不变
- 确保 CUDA 图形捕获和重放的兼容性
- 支持所有现有的注意力状态（DecodeOnly、ChunkedPrefill、SpecDecoding 等）

### 2.3 性能优化
- 保持现有的性能优化策略（MLAPO、DSA CP 等）
- 确保 prefill 和 decode 可以并行执行
- 优化内存分配和重用

## 3. 详细设计

### 3.1 元数据结构设计

#### 3.1.1 AscendSFAPrefillMetadata（新增）
```python
@dataclass
class AscendSFAPrefillMetadata:
    """Prefill-specific metadata for Ascend SFA attention."""
    attn_mask: torch.Tensor
    query_lens: torch.Tensor
    seq_lens: torch.Tensor
    context_lens: torch.Tensor
    input_positions: torch.Tensor
    query_start_loc: torch.Tensor
    block_table: torch.Tensor
    max_query_len: int
    max_seq_lens: int
    sin: torch.Tensor = None
    cos: torch.Tensor = None
```

**字段说明：**
- `attn_mask`: 注意力掩码，用于处理 causal attention
- `query_lens`: 每个请求的 query 长度列表
- `seq_lens`: 每个请求的总序列长度
- `context_lens`: 上下文长度（已缓存的 token 数量）
- `input_positions`: 输入位置编码
- `query_start_loc`: query 起始位置索引（用于计算累计长度）
- `block_table`: block 映射表
- `max_query_len`: 最大的 query 长度
- `max_seq_lens`: 最大的序列长度
- `sin/cos`: RoPE 的正弦/余弦缓存

#### 3.1.2 AscendSFADecodeMetadata（新增）
```python
@dataclass
class AscendSFADecodeMetadata:
    """Decode-specific metadata for Ascend SFA attention."""
    input_positions: torch.Tensor
    block_table: torch.Tensor
    seq_lens: torch.Tensor
    max_seq_lens: int
```

**字段说明：**
- `input_positions`: 输入位置编码（decode 通常只有 1 个 token）
- `block_table`: block 映射表（仅包含 decode 请求的部分）
- `seq_lens`: 每个请求的序列长度（仅包含 decode 请求）
- `max_seq_lens`: 最大的序列长度
- `seq_lens_list`: 序列长度列表（用于 NPU 算子）
- `actual_seq_lengths_q`: 实际 query 长度列表（用于带 query 的场景）
- `attn_mask`: 注意力掩码（可选）
- `sin/cos`: RoPE 的正弦/余弦缓存（仅包含 decode 部分）

#### 3.1.3 AscendSFAMetadata（修改）
在现有结构中添加两个新字段：
```python
class AscendSFAMetadata:
    # ... 现有字段 ...
    
    # 新增字段
    prefill: AscendSFAPrefillMetadata | None = None
    decode: AscendSFADecodeMetadata | None = None
```

**设计要点：**
- `prefill` 和 `decode` 字段都是可选的，根据实际请求类型动态设置
- 当只有 decode 时，`prefill = None`
- 当只有 prefill 时，`decode = None`
- 当混合场景时，两者都存在

```python
def build_decode_metadata(
    self,
    common_prefix_len: int,
    common_attn_metadata: AscendCommonAttentionMetadata,
) -> AscendSFADecodeMetadata:
    """
    构建 decode 阶段的元数据。
    仅处理 decode 请求（通常位于 batch 的前半部分）。
    """
    num_reqs = common_attn_metadata.num_reqs
    query_start_loc_cpu = common_attn_metadata.query_start_loc_cpu
    
    # 提取 decode 相关数据
    actual_seq_lengths_q = query_start_loc_cpu[1: self.num_decodes + 1].tolist()
    max_seq_lens = common_attn_metadata.seq_lens[: self.num_decodes].max().item()
    
    input_positions = common_attn_metadata.positions[: self.num_decode_tokens].long()
    block_table = common_attn_metadata.block_table_tensor[: self.num_decodes]
    
    cos, sin = get_cos_and_sin_mla(input_positions, use_cache=True)
    
    return AscendSFADecodeMetadata(
        input_positions=input_positions,
        block_table=block_table,
        seq_lens=common_attn_metadata.seq_lens[: self.num_decodes],
        seq_lens_list=common_attn_metadata.seq_lens[: self.num_decodes].tolist(),
        max_seq_lens=max_seq_lens,
        actual_seq_lengths_q=actual_seq_lengths_q,
        attn_mask=None,  # decode 通常不需要掩码
        sin=sin[: self.num_decode_tokens, ...],
        cos=cos[: self.num_decode_tokens, ...],
    )
```

```python
def _sfa_preprocess_decode(
    self,
    hidden_states: torch.Tensor,
    kv_cache: tuple[torch.Tensor, torch.Tensor, torch.Tensor],
    attn_metadata: M,
    need_gather_q_kv: bool,
    num_input_tokens: int,
) -> tuple[torch.Tensor, torch.Tensor, torch.Tensor, torch.Tensor]:
    """
    Decode 阶段的预处理方法。
    处理 decode tokens 的 QKV 投影和 RoPE。
    
    返回:
        hidden_states: 处理后的 hidden states
        ql_nope: Decode 的 q_nope
        q_pe: Decode 的 q_pe
        q_c: Decode 的 q_c
    """
    # 1. All-gather hidden states（如果需要）
    hidden_states = torch.ops.vllm.maybe_all_gather_and_maybe_unpad(
        hidden_states.contiguous(), need_gather_q_kv
    )
    
    # 2. 提取 KV cache
    k_nope, k_pe = kv_cache[0], kv_cache[1]
    
    # 3. 创建输出张量
    ql_nope = torch.empty(
        (num_input_tokens, self.W_UK_T.shape[0], k_nope.shape[-1]),
        dtype=hidden_states.dtype,
        device=hidden_states.device,
    )
    q_pe = torch.empty(
        (num_input_tokens, self.W_UK_T.shape[0], k_pe.shape[-1]),
        dtype=hidden_states.dtype,
        device=hidden_states.device,
    )
    q_c = torch.empty(
        (num_input_tokens, self.q_lora_rank),
        dtype=hidden_states.dtype,
        device=hidden_states.device,
    )
    
    # 4. 调用 MLA preprocess 算子
    torch.ops._C_ascend.mla_preprocess(
        hidden_states,
        self.wd_qkv,
        self.deq_scale_qkv,
        self.gamma1,
        self.beta1,
        self.wu_q,
        self.qb_deq_scl,
        self.gamma2,
        attn_metadata.cos,
        attn_metadata.sin,
        self.W_UK_T,
        k_nope,
        k_pe,
        attn_metadata.slot_mapping,
        quant_scale0=self.quant_scale0,
        quant_offset0=self.quant_offset0,
        bias0=self.quant_bias_qkv,
        quant_scale1=self.quant_scale1,
        quant_offset1=self.quant_offset1,
        bias1=self.qb_qt_bias,
        ctkv_scale=self.ctkv_scale,
        q_nope_scale=self.q_nope_scale,
        cache_mode="krope_ctkv",
        quant_mode="per_tensor_quant_asymm",
        enable_inner_out=True,
        q_out0=ql_nope,
        kv_cache_out0=k_nope,
        q_out1=q_pe,
        kv_cache_out1=k_pe,
        inner_out=q_c,
    )
    
    return hidden_states, ql_nope, q_pe, q_c
```

```python
def _sfa_preprocess_prefill(
    self,
    hidden_states: torch.Tensor,
    kv_cache: tuple[torch.Tensor, torch.Tensor, torch.Tensor],
    attn_metadata: M,
    need_gather_q_kv: bool,
    num_input_tokens: int,
    num_decode_tokens: int,
) -> tuple[torch.Tensor, torch.Tensor, torch.Tensor]:
    """
    Prefill 阶段的预处理方法。
    处理 prefill tokens 的 QKV 投影和 RoPE。
    
    返回:
        q_c: Preprocess 后的 q_c
        kv_no_split: 原始的 KV 表示
        k_pe: Preprocess 后的 k_pe
    """
    # 1. All-gather hidden states（如果需要）
    hidden_states = torch.ops.vllm.maybe_all_gather_and_maybe_unpad(
        hidden_states.contiguous(), need_gather_q_kv
    )
    
    # 2. 调用 fused_qkv_a_proj
    qkv_lora = self.fused_qkv_a_proj(hidden_states)[0]
    q_c, kv_no_split = qkv_lora.split(
        [self.q_lora_rank, self.kv_lora_rank + self.qk_rope_head_dim],
        dim=-1,
    )
    q_c = self.q_a_layernorm(q_c)
    
    # 3. 提取 prefill 部分
    prefill_kv_no_split = kv_no_split[num_decode_tokens:num_input_tokens]
    prefill_q_c = q_c[num_decode_tokens:num_input_tokens]
    
    # 4. 提取 prefill 的 RoPE 参数
    prefill_input_positions = attn_metadata.prefill.input_positions
    cos, sin = get_cos_and_sin_mla(prefill_input_positions)
    
    # 5. 处理 Q
    prefill_q = self.q_proj(prefill_q_c)[0].view(-1, self.num_heads, self.qk_head_dim)
    prefill_q_pe = prefill_q[..., self.qk_nope_head_dim:]
    prefill_q_nope = prefill_q[..., :self.qk_nope_head_dim]
    prefill_q_pe = self.rope_single(prefill_q_pe, cos, sin)
    
    # 6. 处理 KV
    prefill_slots = attn_metadata.slot_mapping[num_decode_tokens:num_input_tokens]
    prefill_k_pe, prefill_k_c_normed = self.exec_kv_prefill(
        prefill_kv_no_split, cos, sin, kv_cache, prefill_slots
    )
    
    # 7. 提取 k_nope 和 value
    prefill_k_nope, prefill_value = (
        self.kv_b_proj(prefill_k_c_normed)[0]
        .view(-1, self.num_heads, self.qk_nope_head_dim + self.v_head_dim)
        .split([self.qk_nope_head_dim, self.v_head_dim], dim=-1)
    )
    
    return prefill_q_nope, prefill_q_pe, prefill_k_nope, prefill_k_pe, prefill_value
```

```python
def _forward_prefill(
    self,
    q_nope: torch.Tensor,
    q_pe: torch.Tensor,
    k_nope: torch.Tensor,
    k_pe: torch.Tensor,
    value: torch.Tensor,
    kv_cache: tuple[torch.Tensor],
    attn_metadata: M,
) -> torch.Tensor:
    """
    Prefill 阶段的注意力计算。
    使用 ring-MLA 算子进行高效计算。
    """
    assert attn_metadata.prefill is not None
    
    num_tokens = q_nope.size(0)
    attn_output = torch.empty(
        num_tokens, self.num_heads, self.v_head_dim,
        dtype=q_nope.dtype,
        device=q_nope.device
    )
    attn_lse = torch.empty(
        self.num_heads, num_tokens,
        dtype=torch.float32,
        device=q_nope.device
    )
    
    torch_npu.atb.npu_ring_mla(
        q_nope=q_nope,
        q_rope=q_pe,
        k_nope=k_nope,
        k_rope=k_pe,
        value=value,
        mask=attn_metadata.attn_mask,
        seqlen=attn_metadata.prefill.query_lens,
        head_num=self.num_heads,
        kv_head_num=self.num_heads,
        pre_out=None,
        prev_lse=None,
        qk_scale=self.scale,
        kernel_type="kernel_type_high_precision",
        mask_type="mask_type_triu",
        input_layout="type_bsnd",
        calc_type="calc_type_first_ring",
        output=attn_output,
        softmax_lse=attn_lse,
    )
    
    # 计算 context（如果有）
    attn_output, attn_lse = self._compute_prefill_context(
        q_nope, q_pe, kv_cache, self.qk_rope_head_dim, attn_metadata, attn_output, attn_lse
    )
    
    attn_output = attn_output.reshape([num_tokens, self.num_heads * self.v_head_dim])
    return attn_output
```

```python
def _forward_decode(
    self,
    q_nope: torch.Tensor,
    q_pe: torch.Tensor,
    k_nope: torch.Tensor,

    Decode 阶段的注意力计算。
    使用 fused_infer_attention_score 算子进行高效计算。

    decode_meta = attn_metadata.decode
    
    num_tokens = q_nope.size(0)
    
    # 根据配置调整 KV 格式
    if self.enable_kv_nz:
        nz_fmt_last_dim = 16
        k_nope = k_nope.view(

    forward_context = get_forward_context()
    if forward_context.capturing:
        # 图形捕获模式
        # ... 使用 graph params ...
        pass
    else:

        attn_output = torch.ops._C_ascend.npu_fused_infer_attention_score(

            k_nope,
            k_nope,
```

##### forward（重构）
```python
def forward(
    self,
    layer_name,
    hidden_states: torch.Tensor,
    kv_cache: tuple[torch.Tensor, torch.Tensor, torch.Tensor],
    attn_metadata: M,
    need_gather_q_kv: bool = False,
    output: torch.Tensor | None = None,
) -> torch.Tensor:
    assert output is not None, "Output tensor must be provided."
    
    if attn_metadata is None:
        return output.fill_(0)
    
    forward_context = get_forward_context()
    
    # 获取 token 数量
    num_actual_tokens = attn_metadata.num_actual_tokens
    num_decode_tokens = attn_metadata.num_decode_tokens
    num_prefill_tokens = num_actual_tokens - num_decode_tokens
    
    # 创建输出张量
    o_proj_input_shape = (forward_context.num_tokens, self.num_heads * self.v_head_dim)
    o_proj_input = torch.empty(o_proj_input_shape, dtype=hidden_states.dtype, device=hidden_states.device)
    
    # 预处理
    if self.enable_mlapo and num_input_tokens <= MLAPO_MAX_SUPPORTED_TOKENS:
        hidden_states, ql_nope, q_pe, q_c = self._sfa_preprocess_decode(
            hidden_states, kv_cache, attn_metadata, need_gather_q_kv, num_input_tokens
        )
        q, k = self.indexer_select_pre_process(
            x=hidden_states, qr=q_c, cos=cos, sin=sin, need_gather_q_kv=need_gather_q_kv
        )
    else:
        # 统一预处理
        decode_preprocess_res, prefill_preprocess_res = self._sfa_preprocess(
            layer_name, hidden_states, kv_cache, attn_metadata, need_gather_q_kv
        )
    
    # 处理 decode
    if decode_preprocess_res is not None:
        output_decode = self._forward_decode(
            decode_preprocess_res.ql_nope,
            decode_preprocess_res.q_pe,
            decode_preprocess_res.k_nope,
            decode_preprocess_res.k_pe,
            kv_cache[0].shape[1],  # block_size
            attn_metadata,
        )
        o_proj_input[:num_decode_tokens] = output_decode
    
    # 处理 prefill
    if prefill_preprocess_res is not None:
        output_prefill = self._forward_prefill(
            prefill_preprocess_res.q_nope,
            prefill_preprocess_res.q_pe,
            prefill_preprocess_res.k_nope,
            prefill_preprocess_res.k_pe,
            prefill_preprocess_res.value,
            kv_cache,
            attn_metadata,
        )
        o_proj_input[num_decode_tokens:num_actual_tokens] = output_prefill
    
    # O proj
    weight_prefetch_method = get_weight_prefetch_method()
    weight_prefetch_method.maybe_prefetch_mla_or_sla_weight_in_current_stream(
        inputs=self.o_proj.weight,
        dependency=o_proj_input,
        max_size=MAX_O_PROJ_PREFETCH_SIZE,
        linear_layer=self.o_proj,
    )
    
    output[...] = self.o_proj(o_proj_input, is_prefill=prefill_preprocess_res is not None)[0]
    
    return output
```

### 3.4 Indexer 处理方法

#### 3.4.1 indexer_select_pre_process（重构）
```python
def indexer_select_pre_process(
    self,
    x: torch.Tensor,
    qr: torch.Tensor,
    cos: torch.Tensor,
    sin: torch.Tensor,
    need_gather_q_kv: bool = False,
):
    """
    重构 indexer 的预处理逻辑。
    根据是否有 prefill 和 decode，分别处理。
    """
    k_proj, _ = self.wk(x)
    k_proj = torch.ops.vllm.maybe_all_gather_and_maybe_unpad(k_proj, need_gather_q_kv)
    k = self.k_norm(k_proj).unsqueeze(1)
    k = k.view(-1, 1, self.head_dim)
    
    if HAS_TRITON:
        q, _ = self.wq_b(qr)
        q = q.view(-1, self.n_head, self.head_dim)
        
        cos = cos.view(-1, self.qk_rope_head_dim)
        sin = sin.view(-1, self.qk_rope_head_dim)
        q, k = rope_forward_triton(
            q, k, cos, sin, rope_dim=self.qk_rope_head_dim, 
            is_neox_style=self.is_rope_neox_style
        )
    else:
        # 无 Triton 的实现
        # ...
    
    return q, k
```

#### 3.4.2 indexer_select_post_process（重构）
```python
def indexer_select_post_process(
    self,
    x: torch.Tensor,
    qr: torch.Tensor,
    q: torch.Tensor | None,
    k: torch.Tensor,
    kv_cache: tuple[torch.Tensor, torch.Tensor, torch.Tensor],
    attn_metadata: M,
    cos: torch.Tensor,
    sin: torch.Tensor,
    actual_seq_lengths_query: torch.Tensor,
    actual_seq_lengths_key: torch.Tensor,
    need_gather_q_kv: bool = False,
):
    """
    重构 indexer 的后处理逻辑。
    根据 prefill 和 decode 的不同，使用不同的 seq_lengths。
    """
    weights, _ = self.weights_proj(x)
    weights = torch.ops.vllm.maybe_all_gather_and_maybe_unpad(weights, need_gather_q_kv)
    
    key = kv_cache[2]
    block_table = attn_metadata.indexer_block_table_tensor if self.use_offload else attn_metadata.block_table
    
    # 根据是否有 prefill 来选择使用哪个 seq_lengths
    if attn_metadata.prefill is not None:
        actual_seq_lengths_query = attn_metadata.prefill.query_lens
        actual_seq_lengths_key = attn_metadata.prefill.seq_lens
    elif attn_metadata.decode is not None:
        actual_seq_lengths_query = attn_metadata.decode.actual_seq_lengths_q
        actual_seq_lengths_key = attn_metadata.decode.seq_lens_list
    
    # 调用 indexer
    if self.use_torch_npu_lightning_indexer:
        topk_indices, _ = torch_npu.npu_lightning_indexer(
            query=q,
            key=key,
            weights=weights,
            actual_seq_lengths_query=actual_seq_lengths_query,
            actual_seq_lengths_key=actual_seq_lengths_key,
            block_table=block_table,
            layout_query="TND",
            layout_key="PA_BSND",
            sparse_count=2048,
            sparse_mode=3,
        )
    else:
        topk_indices = torch.ops._C_ascend.npu_lightning_indexer(
            query=q,
            key=key,
            weights=weights,
            actual_seq_lengths_query=actual_seq_lengths_query,
            actual_seq_lengths_key=actual_seq_lengths_key,
            block_table=block_table,
            layout_query="TND",
            layout_key="PA_BSND",
            sparse_count=2048,
            sparse_mode=3,
        )
    
    return topk_indices
```

## 4. 代码结构对比

### 4.1 当前 SFA 实现（问题）
```
forward()
├── 统一预处理（混合 decode 和 prefill）
├── 统一的 indexer 处理
├── 统一的注意力计算
└── 统一的 O proj
```

**问题：**
- 逻辑混杂，难以理解和维护
- 无法针对 prefill 和 decode 进行专门优化
- 调试困难

### 4.2 重构后的 SFA 实现（目标）
```
forward()
├── 预处理
│   ├── _sfa_preprocess_decode() [if num_decodes > 0]
│   └── _sfa_preprocess_prefill() [if num_prefills > 0]
├── 注意力计算
│   ├── _forward_decode() [if decode_preprocess_res is not None]
│   └── _forward_prefill() [if prefill_preprocess_res is not None]
└── O proj
```

**优势：**
- 逻辑清晰，职责分明
- 可以针对不同阶段进行优化
- 易于调试和测试
